# Plan: 01-03: Overzicht Deelgebieden Table Extraction

**Phase:** 1 — PDF Parser
**Goal:** Extract the "Overzicht Deelgebieden" table — V/G/E scores per deelgebied per opdracht — the hardest parsing challenge in the project
**Requires:** Plan 01-02 (text extraction pipeline, line grouping, parseSinglePDF skeleton)
**Verification:** Import 1 PDF, verify deelgebiedScores has correct V/G/E values for all 19 columns by comparing with the original PDF

## Context

The "Overzicht Deelgebieden" table is a grid where:
- **Rows** = opdrachten (assignments), grouped by vak (subject)
- **Columns** = 19 deelgebieden (V&A, M&M, INS, O&DW, C&B, 1E&B, P&O, S&O, ORG, I&B, 2E&B, PrCo, VSK, LOB, INFO, DESK, BS, TOW, BH)
- **Cells** = V (Voldoende), G (Goed), E (Excellent), or empty

This table often spans 2-3 pages. The header row (containing column abbreviations) may repeat on each page. PDF.js extracts raw text items with X/Y coordinates — there is no table structure. The column must be reconstructed by matching each cell's X-position to the nearest column header's X-position.

### Locked Decisions
- D-10: This is the hardest step — 19 columns with V/G/E scores
- D-14: Data model must support deelgebied-scores per leerling for Phase 3

### From Research (01-RESEARCH.md)
- Detect header row: line containing >=5 deelgebied abbreviations
- Build columnMap: { 'V&A': xPosition, 'M&M': xPosition, ... }
- Assign scores to columns by nearest-X matching with COLUMN_X_TOLERANCE of 8pt
- Header row may repeat on each page — detect and skip (reuse same columnMap)
- Use `normalizeScore()` from schema.js to convert V/G/E to canonical levels

## Tasks

### Wave 1 — Table Detection and Score Assignment

**Task 01-03-01: Deelgebied table header detection and column map construction**
Requirements: PDF-05, PDF-07, D-10

Add the deelgebied table parsing to `parsers/pdf.js`. This task finds the table and builds the column position map.

**Implement in `parsers/pdf.js`:**

1. `findDeelgebiedSection(lines)` — function that:
   - Scans all lines looking for either:
     - A line containing text "Overzicht Deelgebieden" (case-insensitive) — marks the start of the section
     - A line where >=5 items match known deelgebied labels — this IS the header row
   - Returns the index of the first header row found
   - If no header row found, returns -1 (signals error for PDF-08)

2. `buildColumnMap(headerLine)` — function that:
   - Takes a line (array of text items) from the header row
   - For each item, checks if `item.str.trim().toUpperCase()` matches any `DEELGEBIEDEN[*].label.toUpperCase()`
   - Records: `{ label: xPosition }` for each matched deelgebied
   - Returns the map, e.g., `{ 'V&A': 45.2, 'M&M': 72.8, 'INS': 98.1, ... }`
   - Validates: if fewer than 5 columns detected, log a warning (table may be malformed)

3. `isHeaderRow(line)` — helper predicate that:
   - Returns true if a line contains >=5 text items that match deelgebied labels
   - Used to detect repeated headers on subsequent pages (skip these, don't parse as data)

**Constants:**
```javascript
const DEELGEBIED_LABELS = window.DEELGEBIEDEN.map(d => d.label.toUpperCase());
const MIN_HEADER_MATCHES = 5; // Minimum deelgebied labels to consider it a header row
const COLUMN_X_TOLERANCE = 8; // Points tolerance for column assignment
```

**Verification:**
- Parse a real PDF and log the columnMap
- columnMap should have entries for all (or nearly all) 19 deelgebieden
- X positions should be monotonically increasing (columns go left to right)
- `isHeaderRow()` correctly identifies the header row and repeated headers

**Done:** Column map correctly built from real PDF header row with >=15 of 19 deelgebieden detected.

**Task 01-03-02: Score row parsing and integration into parseSinglePDF**
Requirements: PDF-05, PDF-07, D-10, D-14

Parse the data rows of the deelgebied table and integrate into the main parse flow.

**Implement in `parsers/pdf.js`:**

1. `assignScoreToColumn(item, columnMap)` — function that:
   - Takes a single text item and the columnMap
   - Finds the column whose X-position is closest to `item.x`, within COLUMN_X_TOLERANCE (8pt)
   - Returns the deelgebied label, or null if no column is close enough
   - This handles the core challenge: PDF text items are not aligned to a grid

2. `parseDeelgebiedTable(lines, startIndex)` — function that:
   - Starts from `startIndex` (the header row found by `findDeelgebiedSection`)
   - Builds `columnMap` from the header row
   - Iterates subsequent lines:
     - Skip lines that are header rows (repeated on new pages — use `isHeaderRow()`)
     - Skip empty lines
     - For each data row:
       - The first item (leftmost X) is typically the opdracht/vak name
       - Remaining items are score cells (V, G, E, or empty)
       - For each score item, call `assignScoreToColumn(item, columnMap)` to find which deelgebied it belongs to
       - Call `normalizeScore(item.str)` to convert to canonical level
     - Detect vak grouping: rows where the leftmost item is a vak heading (larger font or distinct formatting) start a new datapunt group
   - Returns:
     - `datapunten`: array of `{ vak, datapunt, scores: { 'V&A': 'goed', ... } }`
     - `deelgebiedScores`: aggregated flat map — for each deelgebied, the LAST non-null score encountered (latest assessment wins)

3. **Integration into `parseSinglePDF(file)`:**
   - After `parseVakSections(lines)`, call:
     ```javascript
     const deelgebiedStart = findDeelgebiedSection(lines);
     let deelgebiedScores = {};
     let datapunten = [];

     if (deelgebiedStart >= 0) {
       const result = parseDeelgebiedTable(lines, deelgebiedStart);
       deelgebiedScores = result.deelgebiedScores;
       datapunten = result.datapunten;
     } else {
       // Per PDF-08: throw specific error
       throw new Error('Overzicht Deelgebieden tabel niet gevonden');
     }
     ```
   - Populate the StudentRecord's `deelgebiedScores` and `datapunten` fields

**Aggregation strategy for `deelgebiedScores`:**
```javascript
// Initialize all 19 deelgebieden as null
const scores = {};
for (const dg of window.DEELGEBIEDEN) {
  scores[dg.label] = null;
}
// For each datapunt, update with latest non-null score
for (const dp of datapunten) {
  for (const [label, level] of Object.entries(dp.scores)) {
    if (level !== null) {
      scores[label] = level;
    }
  }
}
```

**Edge cases to handle:**
- Score cell is a single character (V, G, E) — normal case
- Score cell is empty or whitespace — skip (normalizeScore returns null)
- Score cell contains unexpected text (dash, dot) — normalizeScore returns null, treated as unassessed
- Very narrow columns causing X-position ambiguity — COLUMN_X_TOLERANCE of 8pt should handle most cases; log a warning if a score item doesn't match any column
- Table ends: detect by reaching the end of the document or a section that is clearly not table data (e.g., a page footer or unrelated heading)
