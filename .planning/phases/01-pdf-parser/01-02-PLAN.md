# Plan: 01-02: PDF Parser Core — Header + Feedback Table

**Phase:** 1 — PDF Parser
**Goal:** Parse PDF header fields (naam, leerling-ID, periode) and per-vak feedback tables (opdrachten, status, feed forward)
**Requires:** Plan 01-01 (PDF.js initialized, schema.js loaded, data model defined)
**Verification:** Import 1 sample PDF, verify naam/leerlingId/periode in console, verify vakken with opdrachten/status/feedforward

## Context

PDF.js is initialized (Plan 01-01). The parser must now extract structured data from voortgang-PDFs. This plan handles everything EXCEPT the "Overzicht Deelgebieden" score table (that is Plan 01-03).

Each voortgang-PDF has this structure:
1. **Header area** (top of page 1): "Naam: [name]", "Leerling ID: [id]", "Periode: [period]", "Leerjaar [N]"
2. **Per-vak sections**: Subject heading, followed by opdrachten with status and feed forward text
3. **"Overzicht Deelgebieden" table** (handled in Plan 01-03)

### Locked Decisions
- D-09: Parser must be robust for small layout variations
- D-11: Status values are "Op tijd ingeleverd en wel beoordeeld", "Zelfevaluatie afgerond", or empty

### From Research (01-RESEARCH.md)
- PDF coordinate system: Y=0 at bottom, sort Y descending for top-to-bottom
- PDF.js does NOT guarantee left-to-right order within a line — must sort by X ascending
- Y_TOLERANCE of 3pt for line grouping (calibrate against real PDFs)
- Font size from `transform[0]` to detect section headings
- `page.cleanup()` after each page to free memory

### Data Model (from Plan 01-01)
The parser must produce a partial `StudentRecord`:
```javascript
{
  naam: "Bosker, J.G. (Javier-Andres)",
  leerlingId: "12345",
  periode: "BJ2 Fase 2 DD",
```

## Tasks

### Wave 1 — Core Extraction Engine

**Task 01-02-01: PDF text extraction and line grouping utilities**
Requirements: PDF-02, PDF-07 (partial — foundation for robust parsing)

Implement the core extraction pipeline in `parsers/pdf.js`. This is the foundation for all parsing — extracting raw text items with coordinates and grouping them into logical lines.

**In `parsers/pdf.js`, implement:**

1. `extractAllTextItems(file)` — async function that:
   - Reads the File as ArrayBuffer via `file.arrayBuffer()`
   - Opens with `pdfjsLib.getDocument({ data: arrayBuffer })`
   - Iterates all pages, calling `page.getTextContent()`
   - For each text item, extracts: `{ str, x: transform[4], y: transform[5], width, height, fontSize: transform[0], page: pageNum }`
   - Filters out empty/whitespace-only items
   - Calls `page.cleanup()` after each page
   - Calls `pdf.destroy()` after all pages
   - Returns flat array of all text items sorted by: page ascending, Y descending (top-to-bottom), X ascending (left-to-right)

2. `groupIntoLines(items, tolerance)` — function that:
   - Takes sorted text items and groups by Y-proximity (default tolerance: 3pt)
   - Items within `tolerance` points of each other's Y coordinate belong to the same line
   - Within each line, items are sorted by X ascending (left-to-right) — CRITICAL per research
   - Uses weighted average Y to handle slight baseline shifts within a line
   - Returns array of lines, where each line is an array of text items
   - Lines are ordered top-to-bottom (page 1 first, then page 2, etc.)

3. `lineToText(line)` — helper that joins a line's items into a single string with space separation

**Implementation notes:**
- Group items by page first, then by Y within each page. Do NOT mix Y coordinates across pages.
- When processing multi-page documents, append page 2 lines after page 1 lines (they are separate coordinate spaces).

**Task 01-02-02: Header extraction and vak/opdracht/feedforward parsing**
Requirements: PDF-02, PDF-03, PDF-04, D-09, D-11

Build on the extraction utilities to parse the header and feedback sections.

**In `parsers/pdf.js`, implement:**

1. `extractHeader(lines)` — function that:
   - Scans the first ~30 lines (header is always at the top)
   - Uses regex to find key-value patterns:
     - `Naam:` or `Naam :` followed by the student name
     - `Leerling ID:` or `Leerling-ID:` followed by the ID
     - `Periode:` followed by the period string
     - `Leerjaar` followed by a number
   - Returns `{ naam, leerlingId, periode, leerjaar }` with empty strings for missing fields
   - Robust: handles variable spacing around colons (per D-09)

2. `parseVakSections(lines)` — function that:
   - Identifies vak (subject) section boundaries using font size heuristic:
     - Vak headings have a larger fontSize than body text (check `transform[0]`)
     - If fontSize detection is unreliable, fall back to: a line that does NOT match any known pattern (not a status, not "Feed Forward:", not a score line) and is followed by opdracht content
   - Within each vak section, identifies opdrachten:
     - Opdracht headers are typically numbered or distinctly formatted lines
     - Status detection: check if any line within an opdracht block contains one of the STATUS_STRINGS (per D-11):
       - "Op tijd ingeleverd en wel beoordeeld"
       - "Zelfevaluatie afgerond"
       - If neither found, status is empty string
     - Feed Forward detection: find a line containing "Feed Forward" or "Feedforward" (case-insensitive), then capture ALL following lines as feed forward text until the next structural marker (next opdracht, next vak heading, or "Overzicht Deelgebieden")
   - Returns array of `{ naam: vakName, opdrachten: [{ naam, status, feedForward }] }`

3. `parseSinglePDF(file)` — the main entry point (async):
   - Calls `extractAllTextItems(file)`
   - Calls `groupIntoLines(items)`
   - Calls `extractHeader(lines)` to get naam/leerlingId/periode/leerjaar
   - Calls `parseVakSections(lines)` to get vakken with opdrachten
   - Validates: if `naam` is empty, try extracting from filename pattern `DD-(.+)\.pdf`
   - Validates: if no vakken found, throw Error with message "Geen vakken gevonden in [filename]"
   - Returns a partial `StudentRecord` (deelgebiedScores and datapunten empty — filled by Plan 01-03)

**Status matching strategy (per D-11):**
```javascript
const STATUS_STRINGS = [
  'Op tijd ingeleverd en wel beoordeeld',
  'Zelfevaluatie afgerond',
];
// Match case-insensitively, with tolerance for extra whitespace
function matchStatus(text) {
  const normalized = text.replace(/\s+/g, ' ').trim().toLowerCase();
  return STATUS_STRINGS.find(s => normalized.includes(s.toLowerCase())) || '';
}
```

**Feed Forward text concatenation:**
- Feed forward may span multiple lines due to PDF.js text fragmentation (per Research Pitfall 6)
- Concatenate all lines between "Feed Forward:" marker and the next structural boundary
- Trim whitespace, collapse multiple spaces

**Export `parseSinglePDF` as the public API:**
```javascript
export { parseSinglePDF };
```

Also make it available as a window global for debugging:
```javascript
window.parseSinglePDF = parseSinglePDF;
```
