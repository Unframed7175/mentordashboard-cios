<objective>
Convert the existing `@media (prefers-color-scheme: dark)` block in `src/index.css` to a JS-controllable `body.dark { }` selector, add a new section 24 of CSS for the SettingsPage layout and the dark-mode toggle switch, neutralize the orphan `@media (prefers-color-scheme: dark)` block in the unused `src/App.css` so it cannot become a second theme source if re-imported, and tokenize the theme-relevant neutral-gray inline colors in `KlasTabStrip.tsx` and `ImportPage.tsx` so `body.dark` actually reaches them.

Purpose: POL-01 requires dark mode to apply consistently to all components through CSS custom properties. Two of the cross-AI reviewers flagged that (a) Codex found `src/App.css` still contains a dark media query that would be a latent second theme source if re-imported, and (b) several components hardcode neutral grays inline that `body.dark` cannot override (since inline styles win over CSS selectors). This plan resolves both — the selector swap, the new section 24 classes, the App.css cleanup, and the inline-color tokenization happen together because they all sit in the "CSS foundation" wave and have no cross-plan dependencies.

Output: A modified `src/index.css` containing the `body.dark { }` block, the removed/commented media query, and a new section 24 with 6 new CSS classes. A modified `src/App.css` with the `@media (prefers-color-scheme: dark)` block removed.
