### Dark Mode Token Mapping — body.dark Overrides

The `@media (prefers-color-scheme: dark)` block in `index.css` must be converted to a `body.dark` selector. All tokens below are already declared in the media query; the implementation is a selector swap, not a redesign.

Tokens requiring `body.dark { }` override (complete list from existing media query):

```
--accent, --accent-hover, --accent-light, --accent-border, --accent-text
--bg-page, --bg-surface, --bg-surface-alt, --nav-bg
--text-primary, --text-secondary, --text-muted, --text-faint
--border-default, --border-light
--shadow-xs, --shadow-sm, --shadow-md, --shadow-lg
--status-groen-bg, --status-groen-text
--status-oranje-bg, --status-oranje-text
--status-rood-bg, --status-rood-text
--status-blauw-bg, --status-blauw-text
--status-grijs-bg, --status-grijs-text
```

Implementation rule: Remove (or comment out) the `@media (prefers-color-scheme: dark)` wrapper. Replace with `body.dark { }` using identical token values. The `:root` block remains unchanged as the light-mode default.

### 3. Dark Mode Toggle

Pure CSS toggle switch, no library. Visual specification:

- Container: `<label className="toggle-switch">` with `display: flex; align-items: center; gap: 12px; cursor: pointer`
- Input: `<input type="checkbox" className="sr-only">` (visually hidden, accessible)
- Track: `<span className="toggle-track">` — width 44px, height 24px, border-radius 12px (pill)
  - OFF state: background `var(--border-default)`, border `1.5px solid var(--border-default)`
  - ON state: background `var(--accent)`, border `1.5px solid var(--accent)`
  - Transition: `background var(--transition-base), border-color var(--transition-base)` (200ms ease)
- Thumb: `<span className="toggle-thumb">` inside track — width 18px, height 18px, border-radius 50%, background `#FFFFFF`
  - OFF position: `translateX(2px)`
  - ON position: `translateX(22px)` (track-width 44 - thumb-width 18 - 2px gap = 24px; adjusted to 22px for 1px inner gap each side)
  - Transition: `transform var(--transition-base)` (200ms ease)
- Label text: `<span>Donkere modus</span>` — font-size 14px (0.875rem), weight 500, color `var(--text-primary)`
- Keyboard: `:focus-visible` on the `<input>` must show outline via the `.sr-only` pattern — use `input:focus-visible + .toggle-track` with `outline: 2.5px solid var(--accent); outline-offset: 2px`

Toggle row layout within "Weergave" section:
```
[toggle-switch label]          [toggle track + thumb]
Donkere modus                          ○——
```
Row: `display: flex; align-items: center; justify-content: space-between`

## Dark Mode Implementation Contract

### index.css Changes
1. Remove the `@media (prefers-color-scheme: dark) { :root { ... } }` block
2. Add `body.dark { }` block with identical token values at the same file location (after `:root`)
3. Keep `@media (prefers-reduced-motion: reduce)` block — do not touch

### JavaScript Logic (SettingsPage or App-level hook)
1. On mount: read `'settings'` key from plugin-store
   - If key exists and `theme` is set: apply `theme` value immediately (add/remove `body.dark`)
   - If key absent: read `window.matchMedia('(prefers-color-scheme: dark)').matches` — apply result, do NOT persist (OS preference, not user choice)
2. On toggle: apply `body.classList.add('dark')` or `remove('dark')` immediately — no delay
3. After toggle: save `{ theme: 'dark' | 'light' }` to plugin-store key `'settings'`
