<objective>
Wire app startup and top-level component so that klassenState is loaded from encrypted store before the React tree renders, and ImportPage is mounted as the sole page.

Purpose: Without loadKlassen() at startup, klassenState.klassen is empty when ImportPage processes imports — students would be written to state that never persists. Without the document-level drop guard, dragging a file and missing the dropzone on Windows triggers browser navigation (the Tauri window loads the file as a URL). Without the storage-error-banner DOM element, Phase 12 storage errors from showStorageError() in utils/klassen.ts are silently swallowed.

Output: Updated src/main.tsx (async startup, loadKlassen before render) and updated src/App.tsx (ImportPage as sole child, document drop guard registered, storage-error-banner div present).
</objective>
