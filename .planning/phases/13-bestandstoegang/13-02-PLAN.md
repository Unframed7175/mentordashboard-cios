## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS file system → dropzone | User drops or selects arbitrary files; File objects arrive via HTML5 FileList |
| File bytes → parser | parseSinglePDF/parseExcelFile/applyBackupRestore receive raw file content |
| Parser output → klassenState | Parsed records are written to in-memory state then encrypted to disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-02-01 | Tampering | handlePDFs file input | mitigate | accept=".pdf,.xls,.xlsx,.zip" filter on input; extension check in handleFiles uses .toLowerCase() and routes correctly; non-matching files produce user-visible Dutch error message, not silent acceptance |
| T-13-02-02 | Denial of Service | batch PDF for...of loop | mitigate | per-file try/catch ensures one corrupt PDF does not abort batch; loop always completes |
| T-13-02-03 | Tampering | handleBackup applyBackupRestore | mitigate | applyBackupRestore validates payload structure (checks typeof payload.klassen === 'object' and not Array); invalid ZIP returns success:false with message — no state mutation |
| T-13-02-04 | Information Disclosure | status/error messages | accept | error messages show filename and error text — acceptable for a single-mentor local tool; no network transmission |
| T-13-02-05 | Elevation of Privilege | File API file.arrayBuffer() | accept | runs in renderer process within Tauri sandbox; no path traversal possible via File object API |
