<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local filesystem → Tauri bundler | Config files (tauri.conf.json, Cargo.toml, entitlements.plist) are read by the Tauri build tool; malformed values produce build errors, not security risks |
| entitlements.plist → macOS kernel | The plist is embedded into the signed binary; entitlement keys not validated by a provisioning profile are silently dropped by ad-hoc signing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | src-tauri/entitlements.plist | mitigate | keychain-access-groups must be absent; if present with ad-hoc signing, errSecMissingEntitlement (-34018) causes keychain failure. Acceptance criterion explicitly checks the string is absent. |
| T-15-02 | Tampering | src-tauri/tauri.conf.json bundle.macOS.signingIdentity | accept | signingIdentity "-" is ad-hoc signing; binary has no verified developer identity. Acceptable for internal school tool (D-05/D-06). HTTPS GitHub Releases distribution mitigates MITM. |
| T-15-03 | Elevation of Privilege | bundle.windows.nsis.installMode | mitigate | installMode "currentUser" ensures install to AppData\Local without UAC prompt; changing to "perMachine" would require admin elevation. Acceptance criterion checks value is exactly "currentUser". |
</threat_model>
