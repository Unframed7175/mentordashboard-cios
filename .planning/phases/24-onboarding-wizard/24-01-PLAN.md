Imports:
  import React, { useState, useRef } from 'react';

Props interface:
  interface OnboardingWizardProps { onComplete: (klasId: string) => void; }

Constants:
  STEP_TITLES — index 1..5 strings (index 0 unused):
    1: 'Welkom bij Mentordashboard CIOS'
    2: 'Voortgang PDFs uploaden'
    3: 'Verzuim Excel uploaden'
    4: 'Stage Excel uploaden (BPV)'
    5: 'Klaar om te beginnen'
  STEP_SUBS — matching subtitles per step (index 1..5).

State:
  step: 1|2|3|4|5 (useState<1|2|3|4|5>(1))
  klasId: string | null
  klasNaamInput: string
  pdfsUploaded: number
  processing: boolean
  stepMsg: string
  stepErr: string

Refs: pdfRef, xlsRef, bpvRef (all HTMLInputElement)

Helpers:
  clearFeedback() resets stepMsg and stepErr.
  nextStep() calls setStep(s => Math.min(5, s+1) as any) then clearFeedback().

Step handlers (each guard on processing flag):

handleCreateKlas():
  - Trim klasNaamInput; if empty set stepErr 'Voer een klasnaam in.' and return.
  - setProcessing(true), clearFeedback().
  - Await createKlas(naam). If result && !result.error: call switchActiveKlas(result.id), setKlasId(result.id), nextStep().
  - On error: set stepErr — 'duplicate' error -> 'Klas "${naam}" bestaat al.', otherwise 'Klas aanmaken mislukt.'.
  - Always setProcessing(false).

handlePDFs(files: File[]):
  - Guard: files.length === 0 || processing -> return.
  - setProcessing(true), clearFeedback().
  - Loop: for each file call addStudent(await parseSinglePDF(f)), increment ok counter. Wrap in try/catch; skip failed files.
  - Await saveKlassen().
  - setProcessing(false). If ok > 0: setPdfsUploaded(n => n + ok), setStepMsg(`${ok} PDF${ok > 1 ? 's' : ''} verwerkt`). Else setStepErr('Geen geldige PDFs gevonden.').

handleVerzuimExcel(file: File):
  - Guard: processing -> return.
  - setProcessing(true), clearFeedback().
  - try: mergeVerzuim(await parseExcelFile(file)), await saveKlassen(), setStepMsg('Verzuim verwerkt').
  - catch: setStepErr('Verwerking mislukt: ' + e.message).
  - setProcessing(false).

handleBpvExcel(file: File):
  - Guard: processing -> return.
  - setProcessing(true), clearFeedback().
  - try: const buf = await file.arrayBuffer(); const data = parseBpvExcel(buf); await saveBpvData({ ...(await getBpvData()), ...data }); setStepMsg('BPV-data verwerkt').
  - catch: setStepErr('Verwerking mislukt: ' + e.message).
  - setProcessing(false).

dropProps(handler, exts) factory:
  Returns { onDragOver: e => e.preventDefault(), onDrop: e => { e.preventDefault(); filter files by ext; if length call handler(files); } }.

Render structure:
  Root: div.onboarding-overlay > div.onboarding-card
  Inside card:
    1. Progress indicator: "Stap {step} van 5" paragraph, then a flex row of 5 div bars (height 4px, accent color if n <= step else border-default).
    2. h2 with STEP_TITLES[step], p with STEP_SUBS[step].
    3. Step-conditional content:
       Step 1: label + text input (autoFocus, Enter key triggers handleCreateKlas, width 100%).
       Step 2: div with dropProps(['.pdf']) + onClick triggers pdfRef.current?.click(). Hidden input ref=pdfRef, accept=".pdf", multiple. Inner p shows pdfsUploaded count if > 0 else drop prompt.
       Step 3: div with dropProps(['.xls','.xlsx']) + onClick triggers xlsRef.current?.click(). Hidden input ref=xlsRef, accept=".xls,.xlsx". Inner p shows stepMsg if set else drop prompt.
       Step 4: div with dropProps(['.xls','.xlsx']) + onClick triggers bpvRef.current?.click(). Hidden input ref=bpvRef, accept=".xls,.xlsx". Inner p shows stepMsg if set else drop prompt.
       Step 5: centered div with 🎉 emoji (fontSize 3rem) and confirmation paragraph mentioning klasNaamInput.trim().
    4. Error: if stepErr, render p in var(--status-rood-text).
    5. Action buttons (right-aligned flex row):
       - If processing: small "Verwerken..." span.
       - Step 1: button.btn-primary-accent onClick=handleCreateKlas, disabled when processing||!klasNaamInput.trim(). Label 'Klas aanmaken →' (or 'Aanmaken...' when processing).
       - Step 2: button onClick=nextStep, disabled when pdfsUploaded===0||processing. Class btn-primary-accent when enabled, detail-nav-btn when disabled. Label 'Volgende →'.
       - Steps 3,4: two buttons — 'Overslaan' (class detail-nav-btn) + 'Volgende →' (class btn-primary-accent), both disabled when processing.
       - Step 5: button.btn-primary-accent onClick={() => onComplete(klasId!)} label 'Naar het dashboard →'.

Export: export default OnboardingWizard
