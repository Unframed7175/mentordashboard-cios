## Interaction Contract

| Trigger | Behaviour |
|---------|-----------|
| User types in number input | No immediate action — value updates local state only |
| `blur` on number input | Round to integer (`Math.round`), clamp to min/max, save to plugin-store, call `onNormenChanged` |
| `Enter` key in number input | Programmatically `.blur()` the input — delegates to blur handler above |
| `Escape` key in number input | Revert to last-saved value (same as `NaamInput` pattern) |
| SBC input blur where `sbcDrempel < sblDrempel` | Show `.norm-warning` paragraph below SBC row — no blocking |
| "Herstel standaard" click | Set `confirmingReset = true` — show confirm/cancel inline |
| "Niet herstellen" click | Set `confirmingReset = false` — hide confirm UI |
| "Ja, herstel" click | Call `resetNormen()`, reload state from store, `onNormenChanged()`, `confirmingReset = false` |
| `onNormenChanged` callback fires | App.tsx increments version counter → KlasOverzicht re-renders → `berekenStatus()` reads `getNormenSync()` |

Source: CONTEXT.md D-06, D-07, D-08, D-09, D-10, D-12.

## Accessibility Contract

Per `ui-ux-pro-max` skill (CRITICAL rules):

- Every `<input>` must have an associated `<label>` with matching `htmlFor` / `id`. Use `id="norm-sbl"`, `id="norm-sbc"`, etc.
- `aria-label` is NOT a substitute for visible labels in a settings form — use `<label>` elements.
- Focus ring: `border-color: var(--border-focus)` + `box-shadow: 0 0 0 3px rgba(0,159,227,0.12)` — already defined on `.settings-number-input:focus`.
- Touch targets: `<input>` elements must be at minimum 44px tall — set `min-height: 44px` on `.settings-number-input` if current padding yields less.
- Current `.settings-number-input` padding `0.5rem` top+bottom = 16px + font 14px ≈ 30px total — add `min-height: 44px` to the existing class when section 5 CSS is written, or apply inline.
- Warning paragraph must use `role="status"` if rendered dynamically so screen readers announce it without focus.
