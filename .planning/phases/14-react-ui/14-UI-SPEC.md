# Phase 14 — UI Design Contract

> Visual and interaction contract for Phase 14: React UI.
> Generated by gsd-ui-researcher. Consumed by gsd-planner, gsd-executor, gsd-ui-auditor.
>
> **Design philosophy:** Faithful React port of the existing app.js/index.html.bak visual language.
> No new design language, no shadcn, no external component library. Plain CSS following the
> established klas-tile / status-badge / detail-section class patterns.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, growth-badge margin, chip gap |
| sm | 8px | klas-tile inner gap, ap-form row gap, mini-bar item gap |
| md | 16px | klas-tile padding, klas-grid gap, kpi-strip gap, spider-charts-row gap |
| lg | 24px | detail-section padding, section title margin-bottom (1.5rem = 24px) |
| xl | 32px | App container horizontal padding |
| 2xl | 48px | Empty state padding (vertical), #klassen-leeg top/bottom padding |
| 3xl | 64px | — (not used in this phase) |

Exceptions:
- Touch target minimum 44px height on interactive rows (klas-tile, vak-header, ap-row)
- Header height is 56px (not a multiple of 8 — preserved from existing design)
- App container max-width: 980px, centred with `margin: 0 auto`

**Source:** index.html.bak — klas-tile padding:16px, kpi-strip gap:16px, spider-charts-row gap:16px, klassen-leeg padding:48px 16px.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 | 1.5 | Default body copy, textarea content |
| Label / meta | 12–14px (0.75–0.875rem) | 400 | 1.4 | Status meta, stat-label, sort-btn, kpi-label |

## Interaction Patterns

### View Routing
```
App.tsx state: 'import' | 'klas' | 'detail'
activeStudentId: string | null
```

| Trigger | Action |
|---------|--------|
| Tile click / Enter keydown | `setView('detail'); setActiveStudentId(id)` |
| "← Terug" button | `setView('klas'); setActiveStudentId(null)` |
| "‹ Vorige" / "Volgende ›" | `setActiveStudentId(prevId/nextId)` (stay in detail) |
| Tab click (KlasTabStrip) | `switchActiveKlas(id); setRefreshKey(k+1); setView(students>0 ? 'klas' : 'import')` |
| Successful import (onImportComplete) | `setRefreshKey(k+1); setView('klas')` |

**Source:** D-14-04; app.js showView / showDetail pattern.

### Search
- Controlled `<input>` bound to `zoekTerm` state
- Filter on `student.naam.toLowerCase().includes(zoekTerm.toLowerCase())`
- Renders immediately on each keystroke (no debounce — see app.js line 1427)
- Empty search result copy: `'Geen leerlingen gevonden voor "{zoekTerm}".'`

### Sort
- State: `sortKey: 'naam'|'status'|'verzuim'`, `sortAsc: boolean`
- Toggle direction when same key clicked again
- Default direction: `naam` → ascending (A→Z); `status` / `verzuim` → descending (worst first)
- Active sort button: `.sort-btn.active` styling

### Tile Navigation (keyboard)
- All tiles have `tabindex="0"`
- Enter key activates tile (same as click)
- Focus ring: `outline:2px solid var(--accent-blue); outline-offset:2px`

### Accordion (VakkenSection)
- Click `.vak-header` → toggle `.vak-card.open` class
- Chevron rotates 180deg via CSS `transform`

### Feedback/Actiepunten Panel (collapsible)
- Click `.feedback-header` → toggle `.feedback-body` display
- Chevron rotates on open

### Actiepunten CRUD
| Action | Trigger | Result |
|--------|---------|--------|
| Add | Click "+ Actiepunt toevoegen" | Show inline form (new) |
| Edit | Click "Bewerken" | Replace row with pre-filled form |
| Save | Form submit | `actiepuntenStore.add/update()` → re-render list |
| Delete | Click "Verwijderen" | `actiepuntenStore.remove()` → re-render list |
| Cancel | Click "Wijzigingen annuleren" | Hide form, re-render list |

No confirmation on actiepunt delete (app.js pattern — no confirm call present).

### Modal (KlasModal)
- Open: click "+" in KlasTabStrip
- Close: click "Annuleren" OR click outside overlay OR press Escape
- Submit: click "Klas aanmaken" OR press Enter in klasnaam field
- Validation fires on submit attempt, not on blur

### Notities Auto-save
- `onChange` → debounce 500ms → `student.notitie = value; saveKlassen()`
- Hint text transitions to "Opgeslagen" (green) on save, fades back after 1500ms
