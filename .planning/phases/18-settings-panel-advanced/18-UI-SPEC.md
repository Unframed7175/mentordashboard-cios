## Spacing Scale

Inherited from Phase 17. No new tokens introduced.

| Token | Value | Usage in Phase 18 |
|-------|-------|-------------------|
| xs | 4px | Icon gaps, toggle column gap in table |
| sm | 8px | Row internal padding, threshold row gap, button margins |
| md | 16px | Section internal padding, input group spacing, margin-bottom on threshold group |
| lg | 24px | Gap between section cards |
| xl | 32px | Settings page top/bottom padding |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Deelgebieden table: max-height 420px with `overflow-y: auto` (scrollable at 19 rows). This is a layout constraint, not a spacing token.
- Number inputs (threshold + BPV): width 80px — sized for 3-digit integer values. Not a spacing token.
- BPV progress bar: height 8px (matches existing verzuim segment bar height of 44px is inappropriate here; a compact 8px track fits a single-line section). This departs from the 4px grid — use 8px (xs × 2) which is on the scale.

## Color

Fully inherited from Phase 17. No new color tokens introduced.

| Role | Value (light) | Value (dark) | Usage in Phase 18 |
|------|--------------|--------------|-------------------|
| Dominant (60%) | `#F1F5F9` (`--bg-page`) | `#0B0F1A` | Settings page background |
| Secondary (30%) | `#FFFFFF` (`--bg-surface`) | `#131929` | Section cards, table row odd background |
| Accent (10%) | `#4F46E5` (`--accent`) | `#818CF8` | Toggle ON state, "Herstel standaard" button focus ring, BPV progress bar fill |
| Destructive | `#EF4444` (hardcoded) | `#EF4444` | "Herstel standaard" button — destructive-adjacent (resets all custom config); see Copywriting section for confirmation approach |

Accent reserved for in this phase:
- Active toggle thumb/track in the Actief column (same pattern as dark mode toggle)
- BPV progress bar fill (progress over expected uren)
- Focus ring on: `.dg-naam-input`, `.dg-leerlijn-select`, `.dg-toggle` input, `.settings-number-input`, `.btn-ghost` (Herstel standaard + BPV-uren importeren), `.btn-primary` (Ja, herstel), inline confirmation dismiss button — `outline: 2.5px solid var(--accent); outline-offset: 2px`

"Herstel standaard" button uses `.btn-ghost` styling (not `.btn-primary`) to avoid high-emphasis for a destructive-adjacent reset action. Its destructive intent is communicated via the confirmation dialog, not the button color.

### 1. Deelgebieden & Leerlijnen Section (SET-03 + SET-04)

**Replaces placeholder in SettingsPage.tsx section 3.**

**Section wrapper:** `<section className="detail-section">` with `<h2 className="detail-section-title">Deelgebieden &amp; Leerlijnen</h2>` — existing classes, no change.

**Table container:**
- Class: `.dg-settings-table-wrap`
- CSS: `max-height: 420px; overflow-y: auto; border: 1.5px solid var(--border-default); border-radius: var(--radius-md);`
- Contains a `<table className="dg-settings-table">` with `width: 100%; border-collapse: collapse`

**Table header row (`<thead>`):**
- 4 columns: **Naam** | **Leerlijn** | **Actief** | (empty — reserved for future)
- Header cells: `<th>` using the label style — font-size 11px, weight 700, uppercase, `letter-spacing: 0.07em`, color `var(--text-faint)`, background `var(--bg-surface-alt)`, padding `8px`, `border-bottom: 1.5px solid var(--border-default)`, `position: sticky; top: 0; z-index: 1`
- Column widths: Naam auto (flex), Leerlijn 160px, Actief 68px

**Table rows (`<tbody>`, 19 rows):**
- Class: `.dg-settings-row`
- CSS: `border-bottom: 1px solid var(--border-light); font-size: 0.875rem`
- Last row: `border-bottom: none`
- Even rows: background `var(--bg-surface-alt)` — matches `.dg-matrix tr:nth-child(even)` pattern
- Odd rows: background `var(--bg-surface)`
- Hover: background `var(--accent-light)` on `td` — matches `.dg-matrix tr:hover td` pattern

**Naam column (`<td>`, first column):**
- Contains `<input type="text" className="dg-naam-input">`
- CSS for `.dg-naam-input`:
  - `width: 100%; padding: 4px 8px; background: transparent; border: 1.5px solid transparent; border-radius: var(--radius-sm); font-size: 0.875rem; color: var(--text-primary); transition: border-color var(--transition-base), background var(--transition-base)`
  - Default (no focus): `border-color: transparent; background: transparent` — reads like plain text
  - Hover (row hover passes through): cursor text
  - Focus: `border-color: var(--border-focus); background: var(--bg-surface); box-shadow: 0 0 0 3px rgba(79, 70, 229, 0.12)` — same focus ring as `.aanvullend-veld select:focus`
- Apply on: `blur` event AND `Enter` keydown. On `Escape` keydown: revert to last-saved value and blur.
- No debounce — apply is intentional (blur/Enter), not on-keystroke.

**Leerlijn column (`<td>`, second column):**
- Contains `<select className="dg-leerlijn-select">`
- CSS for `.dg-leerlijn-select`: reuse `.aanvullend-veld select` visual spec exactly:
  - `padding: 0.5rem 0.75rem; border: 1.5px solid var(--border-default); border-radius: var(--radius-sm); background: var(--bg-surface); color: var(--text-primary); font-size: 0.875rem; outline: none; cursor: pointer; transition: border-color var(--transition-base), box-shadow var(--transition-base); box-shadow: var(--shadow-xs)`
  - Focus: `border-color: var(--border-focus); box-shadow: 0 0 0 3px rgba(79, 70, 229, 0.12)`
  - Width: fit-content within the 160px column
- Options (in this order): `Lesgeven` | `Organiseren` | `Prof. handelen`
- Apply: `onChange` — immediate, same instant-apply pattern as dark mode toggle
- Option values map to leerlijn keys in `utils/leerlijnen.ts`: `"lesgeven"` | `"organiseren"` | `"profHandelen"`

**Actief column (`<td>`, third column):**
- Contains a toggle switch using the **same `.toggle-switch` / `.toggle-track` / `.toggle-thumb` CSS classes** from Phase 17 — no new toggle CSS needed.
- Reduced size for table context: use CSS override `.dg-toggle` class to scale down:
  - `.dg-toggle .toggle-track`: width 32px, height 18px, border-radius 9px
  - `.dg-toggle .toggle-thumb`: width 12px, height 12px
  - OFF position: `translateX(2px)` | ON position: `translateX(16px)`
- No label text (column header "Actief" serves as label)
- `aria-label` on the input: `"Deelgebied {naam} actief"`
- Apply: `onChange` — immediate

**"Herstel standaard" reset button:**
- Position: below the table, `margin-top: 8px`
- Class: `btn btn-ghost`
- Text: `Herstel standaard`
- Width: `fit-content`
- Action: opens an inline confirmation (see Copywriting + Interaction States)

### 2. Drempelwaarden & BPV-uren Section (SET-05 + SET-06)

**Replaces placeholder in SettingsPage.tsx section 4.**

**Section wrapper:** `<section className="detail-section">` with `<h2 className="detail-section-title">Drempelwaarden &amp; BPV-uren</h2>` — existing classes, no change.

**Subsection: Verzuim drempelwaarden (SET-05)**

Layout: `<div className="settings-threshold-group">` — `display: flex; flex-direction: column; gap: 8px; margin-bottom: 16px`

Each threshold row: `<div className="settings-threshold-row">` — `display: flex; align-items: center; gap: 8px`

- `<label>` text at 14px / 400 / `var(--text-primary)`, min-width 160px
- `<input type="number" className="settings-number-input">`:
  - CSS: `width: 80px; padding: 0.5rem; border: 1.5px solid var(--border-default); border-radius: var(--radius-sm); background: var(--bg-surface); color: var(--text-primary); font-size: 0.875rem; text-align: right; box-shadow: var(--shadow-xs); outline: none; transition: border-color var(--transition-base), box-shadow var(--transition-base)`
  - Focus: `border-color: var(--border-focus); box-shadow: 0 0 0 3px rgba(79, 70, 229, 0.12)`

  - Default value: geoorloofd = `15`, ongeoorloofd = `10`

- Default value: `200` (reasonable default for a half-year BPV block — executor may adjust to actual school policy; see D-14 note)
- Unit label: `"uur"`

BPV Excel import button:
- Class: `btn btn-ghost`
- Text: `BPV-uren importeren`
- Position: below the BPV row, `margin-top: 8px`
- Action: triggers file selection for BPV Excel (separate from verzuim Excel)
- Helper text below button: `<p>` at 14px / `var(--text-muted)` / margin-top 8px: `"Importeer de BPV Excel om gerealiseerde uren per leerling bij te houden."`

## Interaction States

### Naam Column Inline Input (`.dg-naam-input`)

| State | Visual |
|-------|--------|
| Default (read-like) | Border transparent, background transparent — appears as plain text |

| Focus | Border `var(--border-focus)`, background `var(--bg-surface)`, focus ring shadow |
| Editing (value changed, not yet applied) | No visual indicator — apply is explicit (blur/Enter) |
| Applied | No success flash — instant update propagates to dashboard silently |
| Escape pressed | Reverts to last-saved value and blurs |

### Leerlijn Dropdown (`.dg-leerlijn-select`)

| Default | Matches `.aanvullend-veld select` — border `var(--border-default)`, shadow `var(--shadow-xs)` |
| Focus | Border `var(--border-focus)`, focus ring shadow |
| Changed | Applies instantly via `onChange` — no save step |

### Actief Toggle (`.dg-toggle`)

| State | Visual |
|-------|--------|
| OFF | Track: `var(--border-default)`, thumb: white at `translateX(2px)` |
| ON | Track: `var(--accent)`, thumb: white at `translateX(16px)` |
| Transition | 200ms ease — inherited from Phase 17 toggle |
| Focus visible | Track outline: `2.5px solid var(--accent); outline-offset: 2px` |

### "Herstel standaard" Button

| State | Visual |
|-------|--------|
| Default | `.btn-ghost` — background transparent, border `var(--border-default)`, color `var(--text-secondary)` |
| Hover | Background `var(--bg-surface-alt)`, border `var(--accent-border)`, color `var(--text-primary)` |
| Focus visible | `outline: 2.5px solid var(--accent); outline-offset: 2px` |
| Clicked (before confirm) | Inline confirmation replaces button (see below) |

**Inline confirmation pattern (no modal):** When the user clicks "Herstel standaard", replace the button in-place with:
```
<span style="font-size: 0.875rem; color: var(--text-secondary);">
  Alles terugzetten naar standaard?
</span>
<button className="btn btn-ghost" style="font-size: 0.875rem; padding: 4px 8px;">Niet herstellen</button>
<button className="btn btn-primary" style="font-size: 0.875rem; padding: 4px 8px;">Ja, herstel</button>
```
Layout: `display: flex; align-items: center; gap: 8px; flex-wrap: wrap; margin-top: 8px`

"Niet herstellen" restores the original "Herstel standaard" button.
"Ja, herstel" resets all deelgebied config to defaults, then restores the original button.

### Data Flow: Instant-Apply Pattern

All Phase 18 settings use the same instant-apply pattern as the Phase 17 dark mode toggle:
1. User interaction fires a handler (blur/Enter for naam; onChange for leerlijn/actief/threshold/bpv-uren)
2. Handler updates in-memory config state
3. Handler calls `store.set(key, value)` → `store.save()` (must pair both, per Phase 12 pitfall)
4. Downstream components (DeelgebiedenMatrix, SpiderChartCard, berekenStatus, berekenPrognose) re-render with updated config
5. No save button required
