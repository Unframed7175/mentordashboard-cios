**2. `renderKlasTabStrip()`** — per D-01/D-02, renders class tabs dynamically:
- Clear `klasTabs.innerHTML`
- For each class in `window.klassenState.klassen` (use `Object.values()`):
  - Create a `<button>` with `className = 'nav-tab'` + `' active'` if it's the activeKlasId
  - `textContent = klas.naam`
  - On click: call `window.switchActiveKlas(klas.id)`, then `renderKlasTabStrip()`, `updateNavCount()`, re-evaluate leerlijn-toewijzing visibility, and `showView('klas')`
  - Append to strip
- Add "+" button: `className = 'nav-tab'`, `textContent = '+'`, `style.color = '#3b82f6'`, `title = 'Nieuwe klas aanmaken'`
  - On click: call `openNieuweKlasModal()`
  - Append to strip
- If no classes exist: hide `klasTabs` (set `display: none`). Otherwise show it (`display: flex`).

**3. `openNieuweKlasModal()` / `closeNieuweKlasModal()`** — per D-03:
- `openNieuweKlasModal()`:
  - Set modal `display` to `flex` (it uses `align-items: center; justify-content: center`)
  - Clear the name input and error message
  - Focus the name input
- `closeNieuweKlasModal()`:
  - Set modal `display` to `none`
  - Clear input and error

**4. Modal event listeners:**
- `modalAnnuleren.addEventListener('click', closeNieuweKlasModal)`
- `modalAanmaken.addEventListener('click', handleCreateKlas)`
- `nieuweKlasNaam` keydown: Enter triggers `handleCreateKlas()`, Escape triggers `closeNieuweKlasModal()`
- Click on modal backdrop (the outer `#nieuwe-klas-modal` div itself, not its children) closes modal

**5. `handleCreateKlas()`** — per D-03/D-04:
- Read `nieuweKlasNaam.value.trim()`
- If empty: show error "Voer een klasnaam in." in `#nieuwe-klas-error`, focus input, return
- Call `window.createKlas(naam)`
- If result has `error === 'duplicate'`: show error "Er bestaat al een klas met de naam '{naam}'.", focus input, return
- Close modal
- Call `renderKlasTabStrip()`
- Call `updateNavCount()`
- Hide `#klassen-leeg` if visible
- Show leerlijn-toewijzing section: `if (ltSection) { ltSection.style.display = 'block'; renderLeerlijntoewijzing(); }`
- Call `showView('import')` — per D-06, new class goes straight to import view
- Show the main nav and klasTabs

**6. `showEmptyKlassenState()` / `hideEmptyKlassenState()`** — per D-05:
- `showEmptyKlassenState()`:
  - Hide `#import-view`, `#klasoverzicht-view`, `#detail-view`, `#main-nav`
  - Show `#klassen-leeg` (`display: block`)
  - Show `#klas-tabs` with just the "+" button (call `renderKlasTabStrip()` — it will show only "+")
- `hideEmptyKlassenState()`:
  - Hide `#klassen-leeg`
  - Show `#main-nav` (`display: flex`)
- `klassenLeegBtn.addEventListener('click', openNieuweKlasModal)` — the CTA button in empty state

**7. Update `showView(view)`** function:
- After the existing hide-all block, add: `hideEmptyKlassenState()` (ensure main-nav is visible whenever a view is shown)
- Keep existing logic intact. The renderKlasoverzicht() call inside showView('klas') stays for now — Plan 03 replaces it.

```javascript
wisDataBtn.addEventListener('click', () => {
  var activeKlas = window.klassenState.klassen[window.klassenState.activeKlasId];
  var klasNaam = activeKlas ? activeKlas.naam : 'deze klas';
  if (!confirm("Klas '" + klasNaam + "' en alle leerlingdata verwijderen? Dit kan niet ongedaan worden gemaakt.")) return;

  detailStudentId   = null;

    showEmptyKlassenState();

    showView('klas');
  }
});
```
