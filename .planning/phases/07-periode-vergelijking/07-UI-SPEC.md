## Visuals

Primary focal point: the `tfoot` rows of the deelgebied matrix in the detail view.

Phase 7 adds at most two footer rows to the existing `.dg-matrix` table. The comparison (Fase 1 / Fase 2) is the only new visual element. Growth arrows and color badges are appended inside individual score chips — they supplement the chip, they do not replace it. The intent is subtle indication, not alarm.

When only one period exists for a student, the detail view renders identically to Phase 6 — a single tfoot row. The two-row comparison only appears when two distinct `periode` values are found for that `leerlingId`.

## Component Inventory

### 1. Detail View tfoot — Two-row Comparison Layout

**Trigger:** `buildDetailDeelgebieden(student)` called from `showDetail(leerlingId)` when two distinct `periode` values exist for a `leerlingId`.

**Structure (`.dg-matrix tfoot`):**

```
<tfoot>
  <tr>  <!-- Fase 1 row -->
    <td class="cell-naam">{periode1 label}</td>
    <td> <span class="dm-chip score-{o|v|g|e|none}">{score}</span> </td>
    ... (one td per deelgebied)
  </tr>
  <tr>  <!-- Fase 2 row (most recent) -->
    <td class="cell-naam">{periode2 label}</td>
    <td>
      <span class="dm-chip score-{o|v|g|e|none}">{score}</span>
      <span class="growth-{up|down|same}" aria-label="{gestegen|gedaald|gelijk}">{↑|↓|=}</span>  <!-- only when fase1 score is not null -->
    </td>
    ... (one td per deelgebied)
  </tr>
</tfoot>
```

**Single-period fallback:** When only one `periode` exists, `tfoot` renders one row (current behavior). No structural change, no empty second row.

**Row label format:** The actual `student.periode` string from the PDF (e.g., "BJ2 Fase 1 DD", "BJ2 Fase 2 DD"). The label is a verbatim copy of the parsed period string — no truncation, no reformatting.

**Column alignment:** `text-align: center` on all score cells (inherits `.dg-matrix td`). Row label cell: `text-align: left` (inherits `.dg-matrix td.cell-naam`).

**No new HTML containers:** The tfoot modification is internal to the existing `.dg-matrix-wrap` > `.dg-matrix` structure. No wrapping elements, no additional `<div>` layers.

### 2. Growth Badge (`<span class="growth-{up|down|same}">`)

Appended as a sibling to `.dm-chip` inside the Fase 2 tfoot cell only.

| Property | Value |
|----------|-------|
| Element | `<span>` |
| Font size | 0.68rem |
| Font weight | 700 |
| Vertical-align | middle |
| Color (up) | `#16a34a` |
| Color (down) | `#dc2626` |
| Color (same) | `#9ca3af` |
| Character (up) | ↑ (U+2191) |
| Character (down) | ↓ (U+2193) |
| Character (same) | = (U+003D) |
| aria-label (up) | `gestegen` |
| aria-label (down) | `gedaald` |
| aria-label (same) | `gelijk` |
| Background | none — transparent, not a chip |
| Border-radius | none |

The badge is intentionally minimal: text-only, no background, no border. It supplements the existing score chip without competing with it visually. The chip color remains the primary signal; the badge adds directionality.

**Not shown when:** Fase 1 score for that deelgebied is null or "—". In that case no basis for comparison exists.

### 3. Detail Header Meta Line (existing, content update only)

`.detail-student-meta` — the single line below the student name that currently shows period, leerjaar, and traject.

In Phase 7, when two periods exist, this line shows the most recent period label (D-08). The DOM structure and CSS are unchanged. Only the string content rendered by `buildDetailHeader()` changes.

Format: `{meest recente periode label} · {leerjaar} · {traject}` (same separator pattern as existing).

### 4. No New Components

Phase 7 introduces no new page sections, modals, buttons, input controls, or navigation elements. The entire UI change is confined to:
- The `.dg-matrix tfoot` (two rows instead of one)
- The growth badge `<span>` appended inside tfoot cells
- The `.detail-student-meta` string content

## Interaction States

### Growth Badge

The growth badge is display-only. It has no interactive states (no hover, no click, no focus). It is a `<span>` with `pointer-events: none` implied by its purely informational role.

### dg-matrix tfoot Rows

Inherits existing `.dg-matrix tfoot td` style (`background: #f3f4f6; font-weight: 700; border-top: 2px solid #d1d5db`).

Row hover: inherits `.dg-matrix tr:hover td { background: #f9fafb }` — both tfoot rows respond to row hover consistently with tbody rows.

| State | Visual |
|-------|--------|
| Default | `background: #f3f4f6` |
| Row hover | `background: #f9fafb` (inherited — matches tbody row hover) |
| No interaction | No click, no keyboard action on tfoot rows |

### Detail View — No Structural State Change

The detail view navigation (prev/next student arrows, "Terug naar overzicht") and all existing interaction states are unchanged. Phase 7 adds no new interactive controls to the detail view.

## Copywriting Contract

| Element | Copy |
|---------|------|
| tfoot Fase 1 row label | Verbatim `student.periode` string for the earlier period (e.g., "BJ2 Fase 1 DD") |
| tfoot Fase 2 row label | Verbatim `student.periode` string for the most recent period (e.g., "BJ2 Fase 2 DD") |
| Detail header meta — single period | `{periode} · {leerjaar} · {traject}` (unchanged) |
| Detail header meta — two periods | `{meest recente periode} · {leerjaar} · {traject}` (most recent period label shown) |
| Growth badge — up | ↑ (no text label, character is sufficient) |
| Growth badge — down | ↓ (no text label, character is sufficient) |
| Growth badge — same | = (no text label, character is sufficient) |
| Single-period state | No comparison UI, no explanatory text — silence is correct (per D-12) |

Language rule: all copy is Dutch, consistent with existing app labels. Period labels are pass-through strings from the PDF parser — not rewritten or translated.

No empty state copy needed for this phase: the single-period fallback renders the existing tfoot silently, and the two-period comparison renders automatically when data is present. No new user-facing error states are introduced.
