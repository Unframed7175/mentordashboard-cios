**Stap A — crypto.rs aanmaken:**

Maak src-tauri/src/crypto.rs aan met de volgende structuur (in volgorde):

1. Imports:
   use tauri::AppHandle;
   use tauri_plugin_secure_storage::SecureStorageExt;
   use aes_gcm::{aead::{Aead, AeadCore, KeyInit, OsRng}, Aes256Gcm, Key, Nonce};
   use base64::{engine::general_purpose::STANDARD, Engine};

2. Constante: const KEY_IDENTIFIER: &str = "nl.cios.mentordashboard.key";

3. Interne helper get_or_init_key(app: &AppHandle) -> Result<Vec<u8>, String>:
   - Roep app.secure_storage().get_item(KEY_IDENTIFIER) aan
   - Bij Ok(response): STANDARD.decode(response.VELDNAAM) — gebruik de werkelijke veldnaam gevonden in Taak 1 (niet hardgecodeerd .value als dat verkeerd bleek)
   - Bij Err(_): genereer nieuw AES-256 key via Aes256Gcm::generate_key(&mut OsRng), encode als Base64, sla op via storage.set_item(KEY_IDENTIFIER, &key_b64), retourneer key bytes

4. Interne helper get_key(app: &AppHandle) -> Result<Vec<u8>, String>:
   - Roep app.secure_storage().get_item(KEY_IDENTIFIER) aan
   - Bij Ok: decode en retourneer
   - Bij Err: retourneer Err("Sleutel niet beschikbaar — neem contact op met beheerder".to_string())

5. pub async fn encrypt_klassen(app: AppHandle, plaintext: String) -> Result<String, String>:
   - key_bytes via get_or_init_key
   - Key::<Aes256Gcm>::from_slice(&key_bytes)
   - Aes256Gcm::new(key)
   - nonce via Aes256Gcm::generate_nonce(&mut OsRng) — ALTIJD vers per aanroep
   - cipher.encrypt(&nonce, plaintext.as_bytes())
   - combined = nonce.to_vec() + ciphertext bytes
   - Ok(STANDARD.encode(&combined))

6. pub async fn decrypt_klassen(app: AppHandle, ciphertext: String) -> Result<String, String>:
   - key_bytes via get_key
   - decoded = STANDARD.decode(&ciphertext)
   - Valideer decoded.len() >= 12, anders Err("Ciphertext too short")
   - split_at(12) → nonce_bytes + ct
   - cipher.decrypt(Nonce::from_slice(nonce_bytes), ct)
   - String::from_utf8(plaintext_bytes)

7. #[cfg(test)] mod tests blok met encrypt_decrypt_roundtrip test:
   - Gebruik een fixed [0u8; 32] test key (geen keychain nodig in unit test)
   - Key::<Aes256Gcm>::from_slice(&key_bytes)
   - Aes256Gcm::new(key)
   - nonce via Aes256Gcm::generate_nonce(&mut OsRng)
   - Verifieer dat decrypt(encrypt(plaintext)) == plaintext

Annoteer beide pub async functions met #[tauri::command].

| Grens | Beschrijving |
|-------|-------------|
| TypeScript → Rust (invoke) | plaintext JSON van klassen.ts wordt doorgegeven aan encrypt_klassen; Rust vertrouwt de inhoud niet — behandelt als opaque bytes |
| Rust → OS keychain | AES-sleutel gaat uitsluitend via tauri-plugin-secure-storage; sleutel verlaat Rust nooit |

## STRIDE Threat Register

| Threat ID | Categorie | Component | Dispositie | Mitigatie |
|-----------|-----------|-----------|------------|-----------|
| T-12-01 | Information Disclosure | AES-sleutel opslag | mitigate | Sleutel opgeslagen in OS keychain (Windows Credential Manager) via KEY_IDENTIFIER 'nl.cios.mentordashboard.key'; nooit in store.json of in TS-layer (D-12-03) |
| T-12-02 | Tampering | Nonce hergebruik | mitigate | Aes256Gcm::generate_nonce(&mut OsRng) op elke encrypt-aanroep — nooit gecached of hergebruikt |
| T-12-03 | Tampering | Ciphertext integriteit | mitigate | AES-GCM voegt 16-byte authentication tag toe; cipher.decrypt() geeft Err bij elke bitwijziging |
| T-12-04 | Elevation of Privilege | Race condition key gen | mitigate | Startup volgorde: await loadKlassen() volledig voor enige saveKlassen(); afgedwongen in Wave 1 plan |
</threat_model>
