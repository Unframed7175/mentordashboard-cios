## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-04 | Tampering | FeedbackModal textarea → buildMailtoUrl | mitigate | buildMailtoUrl (in utils/feedback.ts) applies encodeURIComponent to the full body — covers user-typed description; implemented in Plan 28-01 |
| T-28-05 | Information Disclosure | file.name in setLastImport | accept | File name is visible only in a mailto: URL the user explicitly sends — no server-side collection; user controls what they send |
| T-28-06 | Elevation of Privilege | open() from plugin-opener | accept | open() is sandboxed to mailto: protocol — capability declared as opener:default in capabilities/default.json; user must confirm in OS email client before any data is sent |
| T-28-SC | Tampering | npm/cargo installs | accept | @tauri-apps/plugin-opener and @tauri-apps/plugin-os installed in Plan 28-01 Task 0 — official Tauri packages |
