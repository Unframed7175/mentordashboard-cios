## Spacing Scale

All values are multiples of 4. The existing codebase uses rem-based spacing inline; this scale formalises the token names. No new CSS custom properties are required for spacing — use these values directly in CSS.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon padding, chip internal gaps, tight row gaps in tables |
| sm | 8px | Gap between inline elements, badge padding, row padding in dense tables |
| md | 16px | Section internal padding (horizontal), card gap in toolbar, settings section rows |
| lg | 24px | Section vertical padding, settings header margin-bottom |
| xl | 32px | Page-level top padding (`.settings-page` uses 32px), section card bottom margin |

## Phase-Specific Contracts

### Spider Chart Refactor (POL-02)

**Goal:** Remove `dangerouslySetInnerHTML`. Refactor `buildSpiderSVG` to return `JSX.Element`. Add axis labels and hover tooltips.

**Function signature change:**
```
// Before (utils/spider.ts)
buildSpiderSVG(axes, scores, fillVar, strokeVar): string

// After
buildSpiderSVG(axes, scores, fillVar, strokeVar): JSX.Element
```

**Axis labels (D-10):**
- Always visible; rendered as `<text>` SVG elements at each axis endpoint.
- Position: endpoint coordinates + offset away from center (8px outward from `maxRadius = 80`).
- Radial offset formula: `labelX = centerX + cos(angle) * (maxRadius + 12)`, `labelY = centerY + sin(angle) * (maxRadius + 12)`.
- Text anchor: `text-anchor="middle"` for all labels (SVG attribute).
- Font size: 10px (SVG `font-size="10"`).
- Fill: `var(--text-secondary)` (CSS variable, applied via SVG `fill` attribute).
- Content: `axis.label` value (the custom display label, not the score key).
- No truncation — abbreviated labels from schema are already short (≤6 chars).

**Hover tooltips (D-11):**
- Trigger: `onMouseEnter` / `onMouseLeave` on each axis endpoint hit area.
- Hit area: `<circle>` SVG element at each data point, `r="6"`, `fill="transparent"`, `stroke="none"`, `cursor="pointer"`.
  - The circle sits at the data polygon vertex (score position), not the axis endpoint.
  - This provides a small but reachable hover target on the actual scored position.
- Tooltip state: `useState<{ axisIndex: number; x: number; y: number } | null>(null)` in `SpiderChartCard`.
- Tooltip content: `"{axis.label}: {scoreDisplay}"` where `scoreDisplay` is the capitalised Dutch label:
  - `onvoldoende` → "Onvoldoende"
  - `voldoende` → "Voldoende"
  - `goed` → "Goed"
  - `excellent` → "Excellent"
  - `null` → "Geen score"
- Tooltip rendering: absolutely positioned `<div>` sibling to the `<svg>` inside `.spider-card`, not inside the SVG itself. The `.spider-card` wrapper needs `position: relative`.
- Tooltip CSS:
  - `position: absolute`
  - `background: var(--bg-surface)`
  - `border: 1.5px solid var(--border-default)`
  - `border-radius: var(--radius-sm)`
  - `padding: 4px 8px` (xs/sm tokens)
  - `font-size: 12px` (body role)
  - `color: var(--text-primary)`
  - `box-shadow: var(--shadow-md)`
  - `pointer-events: none`
  - `white-space: nowrap`
  - `z-index: 10`
  - `transform: translate(-50%, -100%)` + `top` / `left` set from SVG coordinate mapped to DOM position.
- Tooltip visibility: no debounce needed for mouse hover on desktop (Tauri app, no touch).
- Transition: `opacity` 0→1 over `var(--transition-fast)` (150ms) for appear; immediate hide on `onMouseLeave`.

**Security note:**
- Remove the existing script-tag strip comment block in `SpiderChartCard.tsx` when removing `dangerouslySetInnerHTML`.
- The JSX approach eliminates the SVG injection surface entirely. Update or remove the security comment at line 41–44 of `SpiderChartCard.tsx`.

**SVG viewBox:** Keep `160×160` rendered / `200×200` viewBox. Do not change chart proportions.
