## Score-telling Row — Exact Layout Contract

### HTML Structure

```tsx
{status.kleur !== 'grijs' && (
  <div className="score-telling" aria-label={scoreAriaLabel}>
    {trend === 'op' && (
      <span className="trend-pijl trend-op" aria-hidden="true" />
    )}
    {trend === 'neer' && (
      <span className="trend-pijl trend-neer" aria-hidden="true" />
    )}
    <span className="score-telling-tekst">
      {status.prognose.totaalVoldoendeOfHoger}/{totaalDeelgebieden} ≥V
      {' · '}
      {status.prognose.totaalOnvoldoende} O
    </span>
  </div>
)}
```

**Rationale for separate elements (Claude's discretion):**
- Arrow `<span>` is `aria-hidden="true"` — the parent `<div>` carries the full `aria-label` so screen readers read "Trend omhoog: 14 van 19 voldoende, 1 onvoldoende" (see Copywriting below).
- Score text is a separate `<span>` so the arrow can be absolutely sized via CSS without affecting text flow.
- The wrapper `<div className="score-telling">` uses `display: flex; align-items: center; gap: 4px` to place arrow and text on the same line.

### CSS Classes

```css
/* Score-telling wrapper row */
.score-telling {
  display: flex;
  align-items: center;
  gap: 4px;
  font-size: 0.75rem;    /* 12px */
  font-weight: 400;
  line-height: 1.4;
  color: var(--text-muted);
  /* padding-left: inherit from .klas-tile-naam (0.375rem / 6px) —
     do NOT declare here; executor must match the existing .klas-tile-naam
     indentation from src/index.css rather than introduce a non-4pt value
     into the design contract. */
}

/* CSS border-trick arrow — up */
.trend-pijl {
  display: inline-block;
  flex-shrink: 0;
  width: 0;
  height: 0;
  /* No margin-top declared — rely on align-items: center on the flex parent.
     Executor must verify visual centering in-browser against 12px Industry text
     and apply a minimal adjustment (typically 1–2px) only if needed.
     Do not introduce a new spacing contract value here. */
  vertical-align: middle;
}

.trend-op {
  border-left:   4px solid transparent;
  border-right:  4px solid transparent;
  border-bottom: 5px solid var(--rag-groen); /* #22C55E */
}

.trend-neer {
  border-left:   4px solid transparent;
  border-right:  4px solid transparent;
  border-top:    5px solid var(--rag-rood);  /* #EF4444 */
}
```

**Indentation note:** `.klas-tile-naam` in `src/index.css` already uses `padding-left: 0.375rem` (6px). This is a pre-existing non-4pt value in the codebase. The executor must apply the same `padding-left: 0.375rem` to `.score-telling` to maintain visual alignment with the tile name — but this value is inherited from existing code, not introduced by this phase. It is not declared in the spacing scale table above for that reason.

**Triangle sizing rationale (Claude's discretion):**
- Score text is 12px at weight 400 in Industry (a condensed, cap-heavy font with slightly tight x-height). Effective visual midline sits ~5px from the baseline.
- `border-left/right: 4px` → triangle base width = 8px.
- `border-bottom/top: 5px` → triangle height = 5px.
- 8×5px triangle has equal visual weight to a 12px character stroke at weight 400. This fulfils D-06 ("same visual weight as score text").
- Vertical centering: `align-items: center` on the flex parent handles this. No `margin-top` is declared in the contract — executor verifies in-browser and applies the smallest needed offset without declaring a new spacing value.

**No dark mode override needed:** `--rag-groen` and `--rag-rood` are stable across modes.

## Copywriting Contract

### Score-telling Text Format

```
{totaalVoldoendeOfHoger}/{totaalDeelgebieden} ≥V · {totaalOnvoldoende} O
```

- `≥V` — literal string, Unicode U+2265 followed by uppercase V
- `·` — interpunct U+00B7 (NOT a hyphen, NOT a bullet U+2022)
- `O` — uppercase letter O (NOT zero 0, NOT lowercase o)
- Single space before and after the interpunct
- Example: `14/19 ≥V · 1 O`
- Example (no trend): `14/19 ≥V · 1 O` (arrow simply absent from DOM)

Source: CONTEXT.md §Specific Ideas.

### Accessibility aria-label

The `.score-telling` wrapper `<div>` receives a dynamic `aria-label` built from the data:

| State | aria-label value |
|-------|-----------------|
| Trend omhoog | `"Trend omhoog: {V} van {totaal} deelgebieden voldoende of hoger, {O} onvoldoende"` |
| Trend omlaag | `"Trend omlaag: {V} van {totaal} deelgebieden voldoende of hoger, {O} onvoldoende"` |
| Geen trend | `"{V} van {totaal} deelgebieden voldoende of hoger, {O} onvoldoende"` |

Example: `"Trend omhoog: 14 van 19 deelgebieden voldoende of hoger, 1 onvoldoende"`

The arrow `<span>` elements carry `aria-hidden="true"` — they are purely decorative.

### Empty / Hidden States

| State | Render |
|-------|--------|
| `status.kleur === 'grijs'` | Entire `.score-telling` div is NOT rendered. No empty space, no placeholder. |
| Both phases present, same rank | Arrow absent from DOM. Score text still shown. |
| Only one phase (no trend comparison possible) | Arrow absent from DOM. Score text still shown if not grijs. |
| `totaalDeelgebieden === 0` | Should not occur (grijs guard fires first), but if it does: show "0/0 ≥V · 0 O" — do not special-case. |

Source: CONTEXT.md D-03, D-09, TREND-04.

## Interaction Contract

### Hover / Focus

The `.klas-tile` hover and focus states are unchanged. The score-telling row inherits tile hover (shadow lift, border accent) without any additional interaction state.

### Click Target

The entire tile remains a single click target (`role="button"`, `tabIndex={0}`, `onClick`, `onKeyDown`). The score-telling row does not introduce nested interactive elements.

### Transition

No new transitions for the score row. Score values are static display (computed from props). The tile's existing `box-shadow` transition (200ms ease) covers hover.

## Props Contract (LeerlingTegel)

New prop added to `LeerlingTegelProps`:

```ts
trend?: 'op' | 'neer' | null;
```

- `undefined` / `null` → no arrow rendered
- `'op'` → `.trend-op` arrow rendered (green, pointing up)
- `'neer'` → `.trend-neer` arrow rendered (red, pointing down)

`totaalDeelgebieden` is derived inside the component from `status.prognose.totaalVoldoendeOfHoger + status.prognose.totaalOnvoldoende`.

## KlasOverzicht trendMap Contract

`KlasOverzicht` must compute a `trendMap: Map<string, 'op' | 'neer' | null>` via `useMemo`:

```
For each student in activeStudents:
  records = getAllRecordsForStudent(student.leerlingId)  // oldest-first
  if records.length < 2: trendMap.set(id, null)
  else:
    fase1Status = berekenStatus(records[0])
    fase2Status = berekenStatus(records[records.length - 1])
    if fase1Status.kleur === 'grijs' OR fase2Status.kleur === 'grijs': null
    else:
      rank1 = STATUS_VOLGORDE[fase1Status.kleur]
      rank2 = STATUS_VOLGORDE[fase2Status.kleur]
      if rank2 > rank1: 'op'
      if rank2 < rank1: 'neer'
      else: null
```
