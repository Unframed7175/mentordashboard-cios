## Blocker — ONB-08: First-run detection broken for zero-student abort path

**File:** `src/App.tsx` line 14
**Current code:**
```typescript
Object.values(klassenState.klassen).some((k: any) => k.students?.length > 0)
  ? 'import' : 'onboarding'
```

**Problem:** A user who creates a klas in step 1 but aborts before importing PDFs ends up with `students: []`. On every subsequent launch, `students.length > 0` returns false → wizard re-runs.

**Fix:** Persist a dedicated `onboardingCompleted: boolean` flag in the store. Set it to `true` in `handleOnboardingComplete`. Check it (alongside or instead of student count) in `App.tsx`.

**Requirement:** ONB-08 — "Wanneer al klassen bestaan, toont de app geen wizard meer bij opstarten."
