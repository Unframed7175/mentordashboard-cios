# Milestone v2.2: Onboarding, Export & Data Completeness

**Status:** ✅ SHIPPED 2026-05-20
**Phases:** 20–24
**Total Plans:** 9 (+ 1 gap closure)
**Timeline:** 2026-05-19 → 2026-05-20

## Overview

Mentors can now set up the app from scratch through a guided 6-step onboarding wizard, print mentorgesprekverslagen as A4 PDF, view BPV stage hours broken down by placement, and track Rekenen & Nederlands results with their own norm. The drag-and-drop file import bug from Phase 16 UAT is fixed. The PDF deelgebied table parser was hardened to handle Unicode dash characters (U+2010) and multi-page page-header rows.

## Phases

### Phase 20: Drag-and-Drop Fix

**Goal**: Mentor kan bestanden slepen op het importveld en de app verwerkt ze correct — PDF, .xls en zip worden allemaal herkend zonder dat de browser navigeert weg van de pagina
**Depends on**: Phase 19
**Requirements**: BUG-01
**Plans**: 1 plan

Plans:
- [x] 20-01-PLAN.md — Set dragDropEnabled:false in tauri.conf.json; bump Cargo.toml to 2.2.0

**Details:**
Added `"dragDropEnabled": false` to `app.windows[0]` in `src-tauri/tauri.conf.json`. Tauri 2 registers a native OS-level drag-drop handler that consumed OS drag events before the browser's HTML5 DataTransfer pipeline ran, leaving `e.dataTransfer.files` always empty. The document-level `preventDefault` guards in `App.tsx` were preserved (unrelated safeguard against accidental drops outside the dropzone).

## Milestone Summary

**Bonus fix (UAT-driven):**
- PDF deelgebied parser hardened — Unicode dash (U+2010 HYPHEN) was being rejected by `startsWith('-')` check (ASCII U+002D). Fixed with regex covering all Unicode dash variants. Also added pending-buffer pattern to discard page-header rows on multi-page tables.

**Decimal Phases:** None

**Key Decisions:**
- `dragDropEnabled: false` in `tauri.conf.json` — single-line fix restoring HTML5 DataTransfer; document-level `preventDefault` guards preserved (unrelated)
- Print CSS uses `#root > *` not `body > *` — React mounts inside `#root`; hiding `body > *` would make `.print-target { display: block !important }` ineffective on a hidden parent
- `@page { size: A4 landscape }` applied to whole document — named `@page` per-element rotation not supported in Tauri WebView2
- BPV parser uses `logboek` keyword — real filename is "Logboek_voortgang_*", not "BPV_*"
- `onboardingCompleted` flag in store — replaces `students.length > 0` as wizard-skip signal; survives zero-PDF abort path

**Issues Resolved:**
- BUG-01: Drag-drop import was silently failing (Tauri native intercept consuming OS events)
- ONB-08: Wizard re-ran on every launch after zero-PDF abort
- PDF parser: Student names and partial vak names appearing as datapunten rows (page headers + Unicode dash mismatch)
- Duplicate UI sections: AanvullendSection and StageSection removed from DetailWeergave (superseded by Rekenen/BPV sections)

**Issues Deferred:**
- RNL-04: PDF extraction for Rekenen/Nederlands — awaiting sample PDF with R&N section
- MIG-01/MIG-02: Integration test fixtures — tests permanently skipped
- BPV-04 loading state: visually indistinguishable from "no data" (cosmetic)
- Nav diagonal stripe: CSS `::after` not rendering in Tauri WebView (cosmetic)

**Technical Debt Incurred:**
- No VERIFICATION.md for any v2.2 phase (UAT files used instead)
- 7 resolved debug sessions not yet archived from `.planning/debug/`
