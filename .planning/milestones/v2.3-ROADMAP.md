## Milestone Summary

**Key Accomplishments:**

- Doorstroomdrempels volledig configureerbaar in settings — mentor kan SBL/SBC/BJ1-normen aanpassen zonder code
- Score-telling op leerlingtegels ("14/19 ≥V · 1 O") + trend-pijl voor fase-vergelijking (↑↓)
- Inline klas hernoemen (double-click) + lege klas verwijderen (hover × knop)
- Bug/feedback rapportage: één klik opent e-mail met OS, versie en console errors automatisch ingevuld
- UI volledig gestreamlind: consistente spacing/typografie, nav-stripe fix, BPV loading state fix, dark mode verfijnd
- In-app helppagina (? knop, 4 secties) + INSTRUCTIES.md + TESTPLAN.md (13 scenario's) voor collega-testers
- GitHub Actions CI: automatische builds op Windows x64 + macOS Apple Silicon bij elke push naar main

**Key Decisions:**

- normalizeRekenScore() apart van normalizeScore() — Rekenen/Nederlands heeft eigen score-formaat
- R&N is student-level (niet period-level) — schrijven naar ALLE records met zelfde leerlingId voorkomt write/read split bij BJ2 studenten met 2 periodes
- CI workflow afgeleid van release.yml — zelfde matrix, maar zonder release-keys (tagName etc.) om ongewenste GitHub Releases te voorkomen
- APPLE_SIGNING_IDENTITY: '-' behouden in CI — ad-hoc signing vereist voor macOS binary zonder Developer cert
- HelpPage static JSX — geen dangerouslySetInnerHTML, geen user-controlled content

**Issues Resolved:**

- BJ2 R&N reactivity bug: DetailWeergave toonde altijd de oudste record (find) terwijl Detail de nieuwste toont
- DoortstroomPrognoseSection crashes op oudere store-schema's (nodigSBC_kern undefined)
- Math.max(undefined, ...) → NaN false-rood op BJ1 Versneld SBC block

**Technical Debt:**

- BPV kolom-matchers vereisen specifiek SomToday BPV-bestand (al meerdere milestones open)
- RNL PDF-extractie wacht op echt PDF met Rekenen/Nederlands sectie (idem)
- RNL-04 en R&N in klasoverzicht-tegels gepland voor toekomstige milestone
