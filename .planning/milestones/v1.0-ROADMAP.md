# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-25
**Phases:** 1–5
**Total Plans:** 10

## Overview

Vijf fases die stap voor stap het dashboard opbouwen. Phase 1 bouwt de PDF-parser — het kritieke hart van het systeem dat voortgang-PDFs per leerling betrouwbaar uitleest. Phase 2 voegt verzuimdata toe via Excel-import en koppelt die aan leerlingen. Phase 3 implementeert de doorstroomnorm-engine die automatisch prognoses berekent. Phase 4 levert het klasoverzicht met rood/oranje/groen per leerling. Phase 5 maakt de detailweergave per leerling voor mentorgesprekken. Elke fase eindigt met een aantoonbaar werkende capability.

## Phases

### Phase 1: PDF Parser

**Goal**: App leest voortgang-PDFs per leerling betrouwbaar in — naam, leerling-ID, periode, opdrachtstatus, feed forward en V/G/E-scores per deelgebied worden correct geëxtraheerd uit alle 19 CSD2A-PDFs
**Depends on**: Nothing (first phase)
**Requirements**: PDF-01, PDF-02, PDF-03, PDF-04, PDF-05, PDF-06, PDF-07, PDF-08
**Plans**: 4 plans

Plans:
- [x] 01-01-PLAN.md — Project scaffold: start.bat, vendored PDF.js (pdfjs-dist 5.5.207), schema.js, data model
- [x] 01-02-PLAN.md — PDF parser core: header extraction + vak/opdracht/status/feedforward
- [x] 01-03-PLAN.md — Overzicht Deelgebieden table extraction (V/G/E scores per deelgebied)
- [x] 01-04-PLAN.md — Import UI: drag-and-drop, file picker, live counter, error display

**Key decisions:**
- pdfjs-dist 5.5.207 vendored as ESM (pdf.min.mjs + pdf.worker.min.mjs) — no CDN dependency, works offline
- workerSrc (not workerPort) — pdfjs 5.x handles {type:'module'} Worker creation internally
- window.* globals pattern — no build step needed, consistent with schema.js/datamodel.js
- COLUMN_X_TOLERANCE=8pt for coordinate-based deelgebied table column assignment

### Phase 2: Excel Import

**Goal**: Mentor kan de verzuim-Excel (.xls) importeren en alle verzuimdata wordt correct gekoppeld aan de leerlingen uit de PDF-import
**Depends on**: Phase 1
**Requirements**: XLS-01, XLS-02, XLS-03, XLS-04
**Plans**: 3 plans

Plans:
- [x] 02-01-PLAN.md — Excel parser (parsers/excel.js) + data model extension (verzuim merge, name matching)
- [x] 02-02-PLAN.md — UI wiring: SheetJS CDN, Excel file picker in index.html, import handler in app.js
- [x] 02-03-PLAN.md — XLSX CDN load guard (gap closure): disable button when SheetJS fails to load

**Key decisions:**
- SheetJS (xlsx-0.20.3) via CDN — no installation, browser-only
- Multi-sheet scanning with name-based scoring (not index 0) — real SomToday exports have metadata at index 0
- Raw array parsing (`header:1`) with dynamic header row detection — column names in real exports differ from plan assumptions
- Float stripping via regex for studentnummer — Excel stores IDs as 248109.0
- 4-strategy name matching: leerlingnummer → full name → achternaam exact → achternaam substring

**Post-execution notes:**
- Gemini cross-AI review correctly predicted all three HIGH/MEDIUM concerns (confirmed by real-world testing)
- "bestandstype wordt niet ondersteund" error during debug was from Windows/browser when PDFs dropped outside drop zone; fixed with document-level dragover/drop prevention

### Phase 3: Doorstroomnorm Engine

**Goal**: App berekent automatisch een doorstroomprognose per leerling op basis van de officiele CIOS-normen, nadat de mentor deelgebieden heeft toegewezen aan leerlijnen
**Depends on**: Phase 2
**Requirements**: NORM-01, NORM-02, NORM-03, NORM-04, NORM-05, NORM-06
**Plans**: 2 plans

Plans:
- [x] 03-01-PLAN.md — berekenPrognose, berekenAllePrognoses, telLeerlijnen (utils/prognosis.js)
- [x] 03-02-PLAN.md — Leerlijn-toewijzing UI + leerlijnen.js persistence (gap closure: NORM-01)

**Key decisions:**
- IIFE wrapping for prognosis.js — prevents local helpers from polluting global scope
- null scores do NOT count as onvoldoende — not-yet-assessed ≠ failed
- Leerlijn-toewijzing persisted in localStorage (`mentordashboard_leerlijnen_v1`) — survives page refresh
- Dynamic mapping in telLeerlijnen() reads localStorage first, falls back to schema.js defaults

## Milestone Summary

**Key Decisions:**
- Browser-only SPA (HTML/CSS/JS) — no build step, no server, no installation for mentor
- PDF per leerling (not class report) — that's how SomToday generates exports
- .xls support required — SomToday exports in old Excel format
- Leerlijn-mapping by mentor — system doesn't know the mapping; mentor does
- Gemini cross-AI review surfaced real-world parsing edge cases before implementation

**Issues Resolved:**
- pdfjs-dist 5.x module worker: workerPort approach broken (sends configure before worker bootstraps); fixed by reverting to workerSrc
- SomToday Excel format: metadata at sheet index 0, non-standard column names, float studentnummer IDs — all fixed with dynamic detection
- Browser PDF drop navigation: document-level dragover/drop prevention added
- XLSX CDN guard: excelChooseBtn now disabled when SheetJS fails to load

**Issues Deferred / Technical Debt:**
- Phase 5 detailweergave verzuim section (DET-02) and prognose tallies (DET-03/04) marked partial — dependent on upstream data quality; requires real CSD2A data to fully validate
- REQUIREMENTS.md was not updated incrementally during execution (all items left unchecked); archived with corrected status
