## Requirements (all marked complete at archive time)

### Bug Fix (BUG)

- [x] **BUG-01**: Bestanden slepen en neerzetten op het importveld werkt correct — PDF, .xls en zip-bestanden worden herkend en verwerkt
  - **Outcome:** Fixed via `dragDropEnabled: false` in `src-tauri/tauri.conf.json`. Phase 20.

### Print-to-PDF Export (EXP)

- [x] **EXP-01**: Mentor kan vanuit de detailweergave een mentorgesprekverslag afdrukken als PDF via de browser print-dialoog
  - **Outcome:** `window.print()` button in DetailWeergave. Phase 21.
- [x] **EXP-02**: Het verslag bevat: leerlingnaam, datum, klasnaam, doorstroomprognose, voortgang per deelgebied (scores + status), verzuimcijfers en actiepunten
  - **Outcome:** Print-only header + full DetailWeergave content in print isolation. Phase 21.
- [x] **EXP-03**: Het verslag is opgemaakt voor A4 papier — geen afgekapte inhoud, correcte paginering
  - **Outcome:** `@page { size: A4 landscape; margin: 1cm 1.5cm }` + compact print CSS. Phase 21.
- [x] **EXP-04**: Een print-knop is zichtbaar in de detailweergave (niet alleen via Ctrl+P)
  - **Outcome:** "Afdrukken" button with `no-print` class (not visible in output). Phase 21.

### BPV Stage Excel Parser (BPV)

- [x] **BPV-01**: App leest echte BPV-uren uit het stage Excel bestand per leerling
  - **Outcome:** Real `parseBpvExcel()` with XLSX parser, column detection, per-row accumulation. Phase 22.
- [x] **BPV-02**: BPV voortgangsindicatie toont werkelijke uren t.o.v. geconfigureerd doelurenaantal
  - **Outcome:** `BpvProgressSection` shows progress bar + per-placement table. Phase 22.
- [x] **BPV-03**: Stage Excel kan worden geïmporteerd via de bestaande importpagina (dropzone + dialoog)
  - **Outcome:** BPV filename routing heuristic `/bpv|stage|praktijk|logboek/i` in ImportPage. Phase 22.
- [x] **BPV-04**: Als geen stage Excel is geïmporteerd, toont de BPV-sectie "Nog geen stage-data" (geen crash)
  - **Outcome:** Empty state message added to BpvProgressSection. Phase 22.
  - **Note:** Loading state is visually identical to empty state — cosmetic debt.

### Rekenen & Nederlands (RNL)

- [x] **RNL-01**: Mentor kan Rekenen voortgang bijhouden — scores apart weergegeven in de detailweergave
  - **Outcome:** RekenenNederlandsSection with 3F/2F/1F dropdown + norm badge. Phase 23.
- [x] **RNL-02**: Mentor kan Nederlands voortgang bijhouden — scores apart weergegeven in de detailweergave
  - **Outcome:** RekenenNederlandsSection with 3F/2F/1F dropdown + norm badge. Phase 23.
- [x] **RNL-03**: Rekenen en Nederlands hebben elk een eigen doorstroomnorm los van de deelgebieden-prognose
  - **Outcome:** `normalizeRekenScore()` maps 1F/2F/3F to pass/fail status independently. Phase 23.
- [x] **RNL-04**: Rekenen/Nederlands scores worden ingelezen uit de bestaande voortgang PDFs (geen apart bestand)
  - **Outcome:** PARTIAL — data model (`rekenResultaat`/`nederlandsResultaat`) and manual-entry UI complete. PDF extraction not implemented — blocked by missing sample PDF with R&N section. Deferred to future milestone.

### Onboarding Wizard (ONB)

- [x] **ONB-01**: Mentor ziet een stap-voor-stap onboarding wizard bij eerste start van de app (geen klassen aanwezig)
  - **Outcome:** App.tsx lazy initialiser checks `onboardingCompleted` flag + student count. Phase 24.
- [x] **ONB-02**: Stap 1 — mentor voert een klasnaam in en maakt de klas aan
  - **Outcome:** Step 1 with klasNaamInput → `createKlas()` + `switchActiveKlas()`. Phase 24.
- [x] **ONB-03**: Stap 2 — mentor uploadt één of meerdere voortgang PDFs voor de klas
  - **Outcome:** Step 2 PDF dropzone; Volgende disabled until `pdfsUploaded > 0`. Phase 24.
- [x] **ONB-04**: Stap 3 — mentor uploadt het verzuim Excel bestand (.xls)
  - **Outcome:** Step 3 .xls/.xlsx dropzone with Overslaan. Phase 24.
- [x] **ONB-05**: Stap 4 — mentor uploadt het stage Excel bestand (BPV-uren); stap is optioneel en kan worden overgeslagen
  - **Outcome:** Step 4 BPV dropzone with Overslaan. Phase 24.
- [x] **ONB-06**: Stap 5 — mentor configureert instellingen (drempelwaarden, deelgebieden/leerlijnen mapping); optioneel, kan worden overgeslagen
  - **Outcome:** Step 5 verzuim drempelwaarden inputs (loadVerzuimDrempels/saveVerzuimDrempels). Phase 24 gap closure.
- [x] **ONB-07**: Na voltooiing van de wizard opent het dashboard direct met de nieuwe klas geladen
  - **Outcome:** Step 6 "Naar het dashboard" → `onComplete(klasId)` → `handleOnboardingComplete` → `switchActiveKlas` → `view='klas'`. Phase 24.
- [x] **ONB-08**: Wanneer al klassen bestaan, toont de app geen wizard meer bij opstarten
  - **Outcome:** Fixed during milestone audit — `onboardingCompleted` boolean persisted in store; loaded in `loadKlassen()` and checked in App.tsx view initialiser. Phase 24 + audit closure.
